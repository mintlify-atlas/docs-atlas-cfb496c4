title: "Article Filters & JavaScript Scripting"
description: "Automate article tagging, scoring, and accept/reject decisions using JavaScript snippets that execute on every feed download in RSS Guard."
---

Article filters are JavaScript snippets that RSS Guard executes automatically whenever articles or feeds are downloaded. Each filter receives the incoming article via the `msg` global, can inspect and mutate its properties â€” marking it important, changing its contents, assigning labels, or adjusting its score â€” and must return a `FilteringAction` value that tells RSS Guard whether to accept, ignore, or purge the article. Filters are managed in the **Article filters** dialog, which also lets you test a filter against existing messages or run it across checked feeds to apply changes permanently.

<Note>
Some attributes (`read/unread/starred` states) are synchronized back to your account's server. For example, marking an article starred in a filter will propagate that change back to a TT-RSS server.
</Note>

## The `filterMessage()` function

Every filter script must expose a function with this exact signature:

```js
function filterMessage() { }
```

The function must return a value from the [`FilteringAction`](#filteringaction-enum) enum. The supported JavaScript standard library conforms to [ECMA-262](https://ecma-international.org/publications-and-standards/standards/ecma-262).

### FilteringAction return values

| Enumerant | Integer | Description |
| :--- | :--- | :--- |
| `Accept` | `1` | Message is accepted and will be added or updated in the database. |
| `Ignore` | `2` | Message is ignored and will **not** be added or updated. Already-existing messages are not removed. |
| `Purge` | `4` | Existing message is purged from the database completely. Behaves like `Ignore` for new incoming messages. |

<Note>
`msg` attributes are synchronized with the remote service even when you return `Purge` or `Ignore`. You can still mutate article properties in a filter that ultimately ignores the article, and those mutations will be synced back to the server.
</Note>

## Global variables

Every filter script has access to the following globals:

| Variable | What it provides |
| :--- | :--- |
| `msg` | The article currently being filtered or modified. |
| `feed` | Information about the feed the article belongs to. |
| `acc` | Account-related information. |
| `app` | Application-wide utilities such as logging and notifications. |
| `run` | Information about the current filter run (counters, indices). |
| `utils` | Utility functions: XML parsing, file I/O, date parsing. |
| `fs` | Filesystem and process-launching actions. |

## Examples

### 1. Whitelist filter with regular expressions

Accept only articles whose titles match at least one pattern in a whitelist; ignore everything else.

```js
/*
 * Accept whitelisted articles based on regular expression filtering.
 */
function filterMessage() {
  // List of regular expressions.
  const whitelist = [
    /ubuntu.+desktop/i,
    /linux.+app/i,
    /\d.billion/i];

  // Accept article if it meets at least one of those regular expressions.
  if (whitelist.some(re => re.test(msg.title))) {
    return Msg.Accept;
  }

  return Msg.Ignore;
}
```

### 2. Mark articles important by keyword

Accept all articles but flag matching ones as important/starred.

```js
/*
 * Mark all whitelisted messages as important.
 */
function filterMessage() {
  const whitelist = [
    /stock/i,
    /crypto/i,
    /market/i];

  if (whitelist.some(re => re.test(msg.title))) {
    msg.isImportant = true;
  }

  return Msg.Accept;
}
```

### 3. Ignore duplicates using Jaro-Winkler similarity

Suppress incoming articles that are very similar to articles already in the database, based on URL similarity.

```js
/*
 * Ignore incoming articles which have very SIMILAR (not same) URL
 * as some article stored in database.
 *
 * Use very high similarity factor 0.05 which ensures that only highly
 * similar URLs are considered "same".
 */
function filterMessage() {
  if (msg.isAlreadyInDatabaseWinkler(Msg.SameUrl, 0.05)) {
    return Msg.Ignore;
  }

  return Msg.Accept;
}
```

### 4. Auto-assign label "AI"

Find or look up the "AI" label and assign it to any matching article. The label must already exist in RSS Guard.

```js
/*
 * Assign label "AI" to all articles matching at least one whitelist condition.
 *
 * Make sure to have label "AI" created beforehand.
 */
function filterMessage() {
  if (/AI|robot|software|hardware/i.test(msg.title)) {
    let id = acc.findLabel("AI");

    if (id) {
      msg.assignLabel(id);
    }
  }

  return Msg.Accept;
}
```

### 5. Desktop notification for breaking news

Show a toast notification and mark the article important whenever a genuinely new breaking-news article arrives.

```js
/*
 * Display desktop notification when totally new breaking news are incoming.
 */
function filterMessage() {
  if (/breaking/i.test(msg.title) && !msg.isAlreadyInDatabaseWinkler(Msg.SameTitle, 0.05)) {
    app.showNotification("Breaking News", msg.title);
    msg.isImportant = true;
  }

  return Msg.Accept;
}
```

### 6. Assign article score from a keywords file

Read a `keywords.txt` file from the user-data folder where each line is `keyword,score`, compute an additive score for the article, and optionally mark it important if the score is high enough.

```js
/*
 * Assign score to articles based on keywords from an external file.
 *
 * Each line in `%data%/keywords.txt` should be:
 *   keyword,score
 *
 * Example:
 *   crypto,50
 *   blockchain,30
 *
 * Scores are additive but capped at 100.
 */
function filterMessage() {
  let keywords = [];

  try {
    let fileContent = utils.readTextFile('%data%/keywords.txt');
    keywords =
        fileContent.split(/\r?\n/)
            .map(line => {
              let parts = line.split(',');
              if (parts.length === 2) {
                return {term: parts[0].trim(), score: Number(parts[1].trim())};
              }
              return null;
            })
            .filter(k => k && !isNaN(k.score));
  } catch (e) {
    app.log('Keywords file missing â†’ default score 0.');
    msg.score = 0;
    return Msg.Accept;
  }

  let totalScore = 0;

  for (let k of keywords) {
    let re = new RegExp(k.term, 'i');
    if (re.test(msg.title) || re.test(msg.contents)) {
      totalScore += k.score;
    }
  }

  // Cap score at 100.
  if (totalScore > 100) {
    totalScore = 100;
  }

  msg.score = totalScore;

  // Optional: mark as important if high score.
  if (msg.score >= 70) {
    msg.isImportant = true;
  }

  return Msg.Accept;
}
```
